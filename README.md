# 발달장애 학생 맞춤 인지 가드레일 (연구 버전)

전문 지식(정확성)과 학생 이해수준을 **분리된 두 가드레일**로 각각 판정하고,
LLM은 그 판정을 통과하는 답을 생성/재작성하는 데만 쓴다.

## 구조
- **전문 가드레일** : 안전 매뉴얼/특수교육학 개론 등 → "정확한 내용인가"
- **학생 가드레일** : 학생 텍스트/그림 키워드 → "이 학생이 이해할 수준인가"
- 두 코퍼스를 섞지 않음(전문용어가 '쉬움' 판정을 망치므로).
- 판정=자체 엔진(CoreAIv2Engine), 생성=OpenAI(선택). LLM이 판단을 대신하지 않음.

## 파일
- `student_profile.py` — 학생 프로필(텍스트/그림 각각 선택)
- `dual_guardrail.py`  — 이중 코퍼스 가드레일(전문/학생 분리 판정)
- `llm_bridge.py`      — OpenAI 연동(전문기반 응답 respond / 쉬운말 변환 simplify)
- `core_ai_v2_engine.py`, `neural_markov_engine.py`, `korean_tokenizer.py` — 엔진
- `app.py`  — 교사용 Streamlit UI (학생입력/전문코퍼스/학습/채팅/검증)
- `demo.py` — UI 없이 동작 확인(※ 데이터 전부 예시용 가짜)

## 실행
```
pip install -r requirements.txt
streamlit run app.py     # UI (사이드바에 OpenAI 키 입력)
python demo.py           # 콘솔 데모 (LLM 없이 판정만)
```

## 사용 흐름
1. 학생 프로필 생성 → 텍스트/그림 키워드 입력
2. 전문 코퍼스 탭에 안전매뉴얼/개론서 추가
3. 학습 탭에서 전문·학생 각각 학습
4. 채팅 탭: 상황 입력 → 전문 기반 + 학생 맞춤 응답(어려우면 자동 쉬운말)
5. 검증 탭: 안내문 판정 + 쉬운 말 변환

## 실제 학생 데이터 주의
- 미성년·발달장애 학생의 발화/그림은 민감 개인정보. 보호자 동의·식별정보 분리 필수.
- 외부 전사(클로바 등) 사용 시 "음성 외부 전송"을 동의서에 명시.
- 데이터가 적으면 판정이 거칠다(코퍼스 양이 성능을 좌우).

## easy-read 베이스 + 학생 보정 (방식 B) — 데이터 희소성 해결
- `easyread_guardrail.py` : easy-read 일반 코퍼스로 베이스 학습(판정 주축),
  학생 개인 단어는 보너스로 얹어 "이 학생한텐 더 쉬움"을 반영.
- 학생 데이터가 적어도 베이스가 판정 토대를 잡아줘 안정적.
- 조정 파라미터(임의 확정 안 함): student_bonus, student_min_count,
  pass_thr/fatal_thr(당신의 -10/-14 철학 유지).
- `demo_easyread.py`로 동작 확인.
- 다음 단계: 실제 서울시 읽기쉬운자료 등 easy-read 자료를 베이스로 모으고,
  실제 학생 단어로 student_bonus 크기를 직접 튜닝.

## 실제 easy-read 베이스 코퍼스 포함
- `easyread_base_corpus.txt` : 장애인사회연구소 '안전한 매일매일'
  (생활안전 편 / 사회재난 편) 쉬운문서에서 본문을 추출·정제한 베이스 코퍼스.
  (출처 자료는 4.16재단·사회복지공동모금회 지원으로 제작된 공개 쉬운문서)
- `demo_base.py` : 이 베이스로 학습 후 쉬운/어려운 문장 판정을 보여주는 데모.
- 검증 결과: 쉬운 문장 PASS, 공문체·전문용어·도메인밖 FATAL로 정확히 구분됨
  (어휘 1552개, 토큰 3898개 — 데이터 희소성 해소).
- 다음 단계: 실제(동의받은) 학생 단어를 student_bonus로 얹어 1인 맞춤.
